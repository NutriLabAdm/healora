# 8. Функциональный прототип и план развития

---

## 8.1. Функциональный прототип Healora Digital Twin

На момент подачи заявки разработан и развёрнут **функциональный прототип** платформы Healora:

| Ресурс | URL |
|--------|-----|
| Продуктивная среда | [healora.ru/digital-twin/](https://healora.ru/digital-twin/) |
| Стенд разработки | [dev.healora.ru/](https://dev.healora.ru/) |
| GitHub | [github.com/NutriLabAdm/healora](https://github.com/NutriLabAdm/healora) |

Прототип представляет собой React SPA (Vite 4) с Node.js/Express и Python/FastAPI бэкендами, взаимодействующими с GigaChat API.

### 8.1.1. Технологический стек (по результатам аудита кода)

| Компонент | Технология |
|-----------|-----------|
| Фронтенд | React 19, Vite 4, React Router 7 |
| Бэкенд #1 | Node.js 20, Express (прокси через nginx :3000) |
| Бэкенд #2 | Python 3, FastAPI (diary API, :3054) |
| AI-ядро | GigaChat API (4-слойный промптинг) |
| Данные | JSON-файлы (9 каталогов), SQLite (дневник) |
| Развёртывание | nginx → Vite build (статику), Node.js API, Python API |
| Сервер | 4 vCPU, 8 GB RAM (Beget) |

## 8.2. Модули прототипа (реализованный код)

### Модуль 1 — Фронтенд (React SPA) `www/dev.healora.ru/src/`

22 React-компонента, точка входа `App.jsx` с маршрутизацией:

| Роут | Компонент | Назначение |
|------|-----------|-----------|
| `/` или `/digital-twin` | `DigitalTwin.jsx` (4561 стр.) | Главный дашборд: таблица параметров DT (50+ атрибутов, 6 разделов: демография, витальные, лаборатории, образ жизни, генетика, медицина), inline-редактирование с историей 7 дней, голосовой ввод (Web Speech API, 8 языков), таймлайн интервенций с DAW-плеером и drag-and-drop, рецепт HEALORA Prescription с QR-кодом, дневник питания, подбор протоколов, выбор целей |

| `/flow` | `DecisionFlowSimulation.jsx` | Интерактивный опросник: ранжирование протоколов по pSuccess, запрос недостающих данных, симуляция |

| `/goals` | `Goals.jsx` | SMART-цели с прогресс-барами (вода, сон, шаги, медитация, чтение), категорийная сводка |
| `/diary` | `NutritionDiary.jsx` | Дневник питания mobile-first: 4 приёма, КБЖУ, фото, голосовой ввод, слайдеры самочувствия |
| б/р (левая панель) | `UserAvatarPanel.jsx` | Каталог цифровых двойников (аватар, имя, ИМТ, АД, звёзды, индикатор риска) |

**Утилиты:** `icons.jsx` (SVG-иконки), `mdParser.js` (Markdown → HTML).

### Модуль 2 — Бэкенд Node.js/Express

Прокси Vite → `localhost:3054` (dev); nginx → `127.0.0.1:3000` (prod).

API-эндпоинты (из анализа кода фронтенда):
- `GET /api/profiles/:id` — загрузка профиля DT
- `PUT /api/profiles/:id` — сохранение изменений
- `POST /api/profiles/:id` — создание (upsert)
- `POST /api/chat` — AI-чат с контекстом профиля
- `GET /api/goals`, `PUT /api/goals` — цели
- `POST /api/diary`, `GET /api/diary` — дневник
- `POST /api/auth/login`, `POST /api/auth/register` — аутентификация

### Модуль 3 — Бэкенд Python/FastAPI (diary API) `backend/main.py` (492 строки)

API на FastAPI с SQLite:
- Таблицы: `diary` (приёмы пищи), `menu_plan` (план меню), `meal_photos` (фото)
- 13 эндпоинтов для CRUD дневника, загрузки фото
- Раздача статики `/images/pers` (аватары профилей)

## 8.4. Use Cases (по коду)

### UC-01: Заполнение цифрового двойника

Пользователь открывает дашборд, выбирает профиль из левой панели (`UserAvatarPanel`). Адресная строка обновляется (`#TEST_002`), система загружает параметры через `GET /api/profiles/:id`. Пользователь редактирует значения inline (двойной клик), или использует голосовой ввод (кнопка микрофона в секции). ИМТ пересчитывается автоматически при изменении веса/роста. Изменения сохраняются через `PUT /api/profiles/:id`, история параметров (7 дней) обновляется.

### UC-02: Получение AI-рекомендации

Пользователь переходит на `/chat`. Система загружает контекст профиля, пользователь задаёт вопрос. `POST /api/chat` отправляет сообщение + контекст в GigaChat (4-слойный промптинг). Ответ возвращается с персонализированной рекомендацией. Дополнительно система предлагает викторину (Quiz) для проверки знаний с начислением звёзд.

### UC-03: Планирование интервенций

Пользователь открывает вкладку «Интервенции» на таймлайне. Правая панель (`InterventionsPanel`) показывает каталог интервенций с evidence level и категориями. Пользователь перетаскивает интервенцию на таймлайн (drag-and-drop). Система размещает трек с точками-днями. Playhead (красный маркер) движется при симуляции (Start/Stop, Speed 1x-10x). Пользователь сохраняет план.

### UC-04: Трекинг питания

Пользователь нажимает «Дневник» на дашборде или переходит на `/diary`. Выбирает день (`◀ Day X ▶`), вносит 4 приёма пищи (завтрак, обед, ужин, перекус). Поиск по 1000+ продуктов из `food_catalog.json`, система подставляет КБЖУ. Добавляет фото, голосовое описание. Регулирует слайдеры самочувствия (вода, энергия, настроение, сон, стресс, пищеварение). Сохраняет через `POST /api/diary` (Python/FastAPI + SQLite).

### UC-05: Подбор протокола через Decision Flow

Пользователь переходит на `/flow`. Система загружает параметры DT, вычисляет pSuccess для протоколов. Показывает Top-3 с вероятностью успеха. Пользователь выбирает протокол, при необходимости система запрашивает недостающие данные (HbA1c, глюкоза, СРБ). После подтверждения протокол добавляется на таймлайн.
