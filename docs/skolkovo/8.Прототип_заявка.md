# 8. Функциональный прототип

Разработан и развёрнут функциональный прототип платформы Healora. Прототип доступен в продуктивной среде по адресу [healora.ru/digital-twin/](https://healora.ru/digital-twin/) и на стенде разработки [dev.healora.ru/](https://dev.healora.ru/), репозиторий — [github.com/NutriLabAdm/healora](https://github.com/NutriLabAdm/healora). Библиотека БАД — [healora.ru/supplements/](https://healora.ru/supplements/).

**Архитектура:** React 19 SPA (Vite 4, React Router 7) на фронтенде, два бэкенда — Node.js/Express (основной API, прокси через nginx) и Python/FastAPI (дневник питания, SQLite). AI-ядро на GigaChat с 4-слойным промптингом и контекстом цифрового двойника.

**Модуль «Цифровой двойник»** — главный дашборд (DigitalTwin.jsx, 4561 строка кода). Таблица с 50+ параметрами здоровья в 6 разделах: демография, витальные показатели, лаборатории, образ жизни, генетика, медицинская история. Каждый параметр имеет колонки «было — текущее — цель — норма» и историю изменений за 7 дней. Inline-редактирование двойным кликом, автосохранение, автоматический пересчёт ИМТ при изменении веса или роста. Голосовой ввод на 8 языках через Web Speech API. Интеграция с каталогом интервенций: ховер на параметре показывает связанные протоколы с impact score, отклонение от цели подсвечивается алертом.

**Модуль «Умный таймлайн»** — планировщик интервенций в формате DAW-плеера. Протоколы располагаются на треках со шкалой времени, красный playhead показывает текущий день. Поддерживает симуляцию с ускорением (1x–10x), переключение вида (день/неделя/фазы). Интервенции перетаскиваются из бокового каталога на треки. Панель управления: Clear, Start/Stop, счётчик дня.

**Модуль «Decision Flow»** — интерактивный опросник подбора протокола терапии. Анализирует параметры цифрового двойника, ранжирует протоколы по вероятности успеха (pSuccess), показывает Top-3 с персонализированной оценкой. При нехватке данных запрашивает недостающие показатели с оценкой влияния на точность подбора. Системный лог отображает все расчёты с временными метками.

**Модуль «AI-Чат»** — полноэкранный диалоговый интерфейс с GigaChat. Имеет контекст выбранного профиля, боковую панель с источниками данных и журналом действий. Поддерживает карточки-задачи со статусом выполнения и звёздными наградами, викторины с вопросами и результатом, быстрые кнопки для типовых запросов.

**Модуль «Дневник питания»** — трекинг питания с четырьмя приёмами пищи (завтрак, обед, ужин, перекус). Встроенный поиск по 1000+ продуктов с автоподстановкой КБЖУ из локальной базы. Загрузка фото приёма, голосовой ввод описания, аудиозапись для консультаций. Слайдеры самочувствия: вода, энергия, настроение, сон, стресс, пищеварение. Данные сохраняются в SQLite через Python/FastAPI бэкенд.

**Модуль «Путь здоровья»** — экран прогресса пользователя с селектором профилей, статистикой (дни в пути, этап, процент выполнения), рекомендациями по трём категориям (ментальное, физическое, образ жизни). Пять индикаторов состояния цифрового двойника: энергия, фокус, стресс, сон, настроение. DeviceDropZone для импорта данных с носимых устройств.

**Модуль «Рецепт HEALORA Prescription»** — генератор персонализированного плана терапии с шапкой (пациент, врач, дата, длительность), селектором шаблона из plan_templates.js, таблицей назначений с периодичностью и расписанием, заметкой врача, QR-кодом со ссылкой на план. Статусы: Active/Stopped/Archived, печать в PDF.

**Модуль «Каталог интервенций»** — боковая панель с библиотекой интервенций, отфильтрованных по категориям (питание, сон, активность, ментальное, медицина, добавки). Evidence level A–D с цветовой индикацией. Карточка протокола содержит название, категорию, бейджи кодов, impact score. Drag-and-drop на таймлайн, попап с полным описанием, регулярностью и источниками.

**Бэкенд-модули:** REST API на Node.js/Express (профили, чат, цели, аутентификация) и Python/FastAPI (дневник питания, SQLite, 13 эндпоинтов, загрузка фото, раздача статики аватаров).

**Данные:** 9 JSON-каталогов — интервенции, протоколы, продукты (1000+), диеты, БАД, предпочтения, ограничения, типы протоколов, шаблоны планов. 13 медицинских практик загружаются через import.meta.glob. Self-contained библиотека 82 монографий БАД с фильтрацией по 8 сценариям.

**CI/CD и стек разработки:** Фронтенд — React 19, Vite 4, React Router 7. Бэкенд — Node.js 20 / Express, Python 3 / FastAPI. Хранилище — JSON-файлы (каталоги) и SQLite (дневник). Сборка — Vite build с разделением по окружениям (development/production через .env с VITE_BASE_PATH). CI/CD реализован по упрощённой схеме: `npm run build` формирует статику в `dist/`, deploy-скрипты (`deploy-dev.sh`) копируют собранные файлы на сервер. Продакшен-маршрут: разработка → коммит в Git → ручной запуск сборки → копирование на сервер через deploy-скрипт. Разработана PlantUML-диаграмма целевого CI/CD-пайплайна (dev_CICD.puml) с этапами: спецификация → разработка → Git → сборка и тестирование → security scan → AI-валидация → Docker-сборка → Kubernetes/облачное развёртывание → мониторинг и логирование.

**Развёртывание:** nginx с Let's Encrypt SSL на сервере 4 vCPU, 8 GB RAM (Beget). Продакшен-сборка (Vite build) статически раздаётся nginx, API-запросы проксируются на Node.js :3000 и Python :3054. Окружение: VITE_BASE_PATH=/digital-twin (production), пустой (development) — управляет корневым путём приложения.

**Документация:** 8 PlantUML-диаграмм (архитектура развёртывания, компоненты React, потоки данных, карта 11 функциональных модулей, архитектура v2, управление контекстом AI, CI/CD пайплайн, RAG-пайплайн). 5 CJM-диаграмм (новый пользователь, активный пользователь, пациент клиники, врач, пилот). Полная спецификация всех экранов (337 строк), архитектурное описание, бэклог, канбан, политика UI/UX, документы по мобильной версии и AI API.
