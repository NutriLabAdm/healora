# Приложение 5. Схема коммерциализации

## 5.1. Основная отрасль применения

> **Рекомендательные системы, персональные помощники и агрегаторы**

Проект Healora относится к классу AI-powered рекомендательных систем в здравоохранении: на основе данных цифрового двойника (антропометрия, анализы, образ жизни, генетика, медицинская история) система подбирает персонализированные нутрициологические протоколы, отслеживает комплаентность и динамику, а при необходимости направляет к профильному специалисту.

## 5.2. Основные направления коммерциализации проекта Healora

### Направление 1. B2B-продажи в частные клиники

| Параметр | Значение |
|----------|----------|
| **Цель** | Удержание пациентов и повышение их комплаентности |
| **Модель** | White-label платформа + API-интеграция с CRM клиники |
| **Каналы** | Прямые продажи через медицинских директоров, участие в профильных выставках (DeepFoodTech, VITA, Здравоохранение) |
| **Целевые клиенты** | Крупные московские сети: МЕДСИ, EMC, «Мать и дитя», «Рассвет», клиники интегративной медицины |
| **Ценность для клиники** | Повышение LTV пациента, снижение оттока (врач видит динамику между визитами), автоматизация первичного скрининга |
| **Модель дохода** | Подписка: 150–300 тыс. руб/мес за инстанс + оплата за активного пациента |
| **Горизонт** | Q4 2026 – пилот в 1–2 клиниках; 2027 – масштабирование до 10+ клиник |

**Цитата:** «Первым направлением коммерциализации нашего проекта будут продажи в частные клиники, целью которых является удержание пациентов и повышение их комплаентности. Мы начнём с крупных московских центров, таких как МЕДСИ, EMC и «Мать и дитя».»

| Параметр | Значение |

### Направление 7. Сопровождение графика терапевтических протоколов (Protocol Concierge)

| Параметр | Значение |
|----------|----------|
| **Цель** | Сквозное ведение пациента по назначенному протоколу с AI-контролем каждого шага + живой нутрициолог для сложных кейсов |
| **Модель** | Пациент получает персональный «календарь протокола»: дни → действия → отметки → AI-корректировка → еженедельный отчёт |
| **Как работает** | |
| 1 | Врач/нутрициолог назначает протокол (TRE_168 + KETO_STD + FD_CAL) |
| 2 | Healora строит timeline: что, когда, как измерять, какие чек-поинты |
| 3 | AI напоминает, отслеживает комплаентность через отметки в чате |
| 4 | При отклонении (пропуск окна питания, срыв КЕТО) — AI корректирует или предлагает альтернативу |
| 5 | Еженедельный дайджест для врача: комплаентность, динамика DT, рекомендации |
| 6 | Сложный кейс — escalation на живого нутрициолога (партнёрская модель) |
| **Модель дохода** | 990–1 990 руб/мес (включено в Premium + надбавка) или 3 000–5 000 руб/курс (42 дня) |
| **Ценность** | Решает главную проблему диетотерапии: 70–80% срывов в первые 1–2 года без поддержки |
| **Бенчмарк** | GLP-1 сопровождение: +30% удержания на терапии при наличии AI-коучинга (данные Noom, 2024) |
| **Горизонт** | Q3 2027 – пилот на 1 протоколе (TRE_168); Q1 2028 – 5+ протоколов |

### Направление 8. AI-скрининг и «второе мнение» для нутрициологов (SaaS-инструмент Pro)

| Параметр | Значение |
|----------|----------|
| **Цель** | Инструмент для профессиональных нутрициологов: AI проверяет план питания, находит противоречия, подтягивает свежие источники |
| **Модель** | B2SMB-подписка для самозанятых и частных нутрициологов |
| **Функции** | — Проверка плана на дефициты и избытки (AI-экспертиза)<br>— Поиск лекарственно-пищевых взаимодействий (TxAgent-подобный модуль)<br>— Рекомендации по уровням доказательности (GRADE)<br>— Библиотека протоколов (десятки протоколов → сотни интервенций)<br>— Генерация отчёта для пациента на понятном языке |
| **Модель дохода** | 1 500–3 000 руб/мес за рабочее место |
| **Ценность** | Нутрициолог работает быстрее в 2–3 раза (AI делает черновик), выше качество за счёт RAG-актуализации |
| **Конкуренция** | Аналогов на российском рынке нет (зарубежные: Healthie, Nutrium — без AI-протоколов) |
| **Горизонт** | Q4 2027 – MVP для 10 бета-тестеров; Q2 2028 – коммерческий запуск |

## 5.4. Научные группы, институты и компании, ведущие аналогичные разработки

| Организация | Разработка | Фокус | Отличие Healora |
|-------------|-----------|-------|-----------------|
| **Harvard / Zitnik Lab** | TxAgent (211 tools, 92.1% accuracy) | Терапевтический reasoning, drug-drug interaction | Нет нутрициологии, нет on-premise, нет русского языка |
| **Google DeepMind** | MedGemma 4B/27B | Медицинское понимание текста и изображений | Фундаментальная модель, не продукт; нет готовых протоколов |
| **Sber AI Lab** | GigaPevt (мультимодальный ассистент) | Общая медицина, RuMedBench | Нет нутрициологии, нет цифрового двойника, нет B2B-модели |
| **Tsinghua AIR** | Agent Hospital (MedAgent-Zero) | Виртуальная больница, AI-доктора | Исследовательский проект; не нутрициология; не РФ |
| **Alibaba / Ant Healthcare** | AntAngelMed (#1 HealthBench) | Open-source медицинская LLM | Фундаментальная модель; не продукт; нет русского языка |
| **Томский ГУ / AIRI** | CLARITY (FSM + LLM routing) | Маршрутизация пациентов, триаж | Не нутрициология; нет DT; нет протоколов |
| **ИТМО / AIRI** | MedSyn (синтетические клинические заметки) | Генерация данных для русского языка | Инструмент, не продукт; не нутрициология |

**Преимущества Healora перед указанными группами:**

1. **Междисциплинарность:** проект сочетает медицинские LLM, нутрициологию, поведенческую психологию и геймификацию — ни одна из перечисленных групп не покрывает все три области одновременно
2. **Продуктовая зрелость:** в отличие от исследовательских прототипов (Yaoshi-RAG, NutriOrion), Healora имеет MVP, план регистрации МИ и B2B-канал
3. **Локализация под РФ:** native Russian + on-premise + 152-ФЗ — обязательное требование для российских клиник, не обеспечиваемое ни одним международным аналогом
4. **Открытая модульная архитектура:** возможность замены LLM-ядра (GigaChat ↔ MedGemma ↔ Qwen) без переписывания бизнес-логики

## 5.5. Рынки

### 5.5.1. Целевые рынки и их ёмкость

#### Рынок РФ (первичный)

| Сегмент | Объём (TAM) | Адресуемый (SAM) | Доступный (SOM) | CAGR |
|---------|------------|-------------------|-----------------|------|
| Частные клиники (B2B) | 5 000+ клиник в РФ | 200 клиник (Москва + СПб + миллионники) | 10–20 клиник (2027) | 12% |
| Пользователи B2C с ИМТ ≥25 | 50 млн чел. | 5 млн (готовы платить за digital health) | 10 000 платящих (2028) | 8–10% |

| Регион | TAM | Обоснование |
|--------|-----|-------------|
