Агент оркестрации

Orchestration Agent — оптимизация метрик SCORE и COMPUTE в мульти-агентной системе

Назначение

Агент оркестрации управляет композицией AI-агентов в пайплайне распознавания и анализа, динамически выбирая набор моделей и глубину обработки для достижения целевого качества (SCORE) при минимальных вычислительных затратах (COMPUTE). Решение принимается на основе контекста: тип блюда, качество фото, требования к точности, доступные ресурсы.

Метрики

SCORE
Качество распознавания (NDI, 0–100)
Взвешенная метрика точности ингредиентов, КБЖУ, уверенности модели
COMPUTE
Вычислительная стоимость (ms / токены)
Латентность + потребление токенов, приведённые к единой шкале

Целевая функция

max( SCORE )  ·  min( COMPUTE )  ⟶  max ( SCORE COMPUTE )
при условии SCORE ≥ 85 · COMPUTE ≤ budget

Динамика COMPUTE при росте активности пользователя

Оркестратор накапливает персональный контекст и сокращает перебор агентов, удерживая SCORE в целевом диапазоне 85–90.

Активность пользователя (LTV)

COMPUTE на запрос по мере роста опыта пользователя

среднее COMPUTE разброс ±1σ уникальные вопросы

Масштабирование платформы

Точки — случайные запросы разных типов (кросс-обучение) · шкала 0–10K

3 причины асимптотического снижения COMPUTE с ростом LTV:
1 Персонализация RAG-контекста — история питания пользователя сужает пространство поиска, уменьшая кол-во агентов для инференса.
2 Профиль Digital Twin — накопленные метаболические параметры позволяют оркестратору точнее выбирать стратегию без перебора.
3 Повторяющиеся паттерны питания — частые блюда кэшируются на уровне пользователя, исключая повторный CV-инференс.
3 причины асимптотического снижения COMPUTE с ростом базы:
1 Кросс-валидация эмбеддингов — чем больше пользователей, тем точнее центроиды пищевых кластеров, меньше ложных срабатываний и повторных инференсов.
2 Кэширование результатов — популярные блюда и комбинации распознаются 1 раз, результат переиспользуется для всех пользователей (hit rate > 60% после 50K юзеров).
3 Обучение оркестратора — RL-политика выбора агентов улучшается с каждым запросом, уменьшая долю дорогих ансамблевых инференсов без потери SCORE.
