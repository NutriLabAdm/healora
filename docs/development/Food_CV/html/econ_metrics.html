Экономика, метрики и оптимизация

Две основные части затрат
R&D (разработка)
  • Обучение моделей
  • Разметка данных
  • Эксперименты
  • Сбор доказательной информации
  • Изучение моделей командой ML
CAPEX (разовые / периодические)
Production (эксплуатация)
  • API LLM или GPU-инференс
  • Embeddings, Vector DB
  • Инфраструктура и мониторинг
  • Поддержка и масштабирование
OPEX (растёт с пользователями)
Главная особенность — себестоимость масштабируется с нагрузкой, а не фиксирована, как в SaaS.
Пример Unit-экономики
1 запрос
≈ 1.55 ₽ (LLM + RAG + инфраструктура)
Пользователь (300 запросов/мес)
≈ 46.5 ₽ себестоимость
Цена 499 ₽/мес
Маржа ~85% (падает при росте активности пользователей)
Цена 1499 ₽/мес
Маржа ~95%
Маржа: один пользователь vs клиентская база
Эффект оптимизации: кэширование типовых вопросов в RAG и оркестрация COMPUTE удерживают маржу на пользователе, а накопление кэша по всей клиентской базе повышает общую доходность платформы.
Ключевые метрики
Стоимость 1k токенов
Базовая стоимость использования модели
Tokens per user
Среднее количество токенов на пользователя в месяц
Cost per session
Себестоимость одной сессии / запроса
ARPU
Средняя выручка на пользователя
LTV / CAC
Ценность пользователя к стоимости привлечения
Основные способы оптимизации
Кэширование
Кэш ответов, эмбеддингов, частых запросов
Снижение токенов
Оптимизация промптов, сжатие контекста, фильтрация
Routing между моделями
Использование small-моделей для простых задач, big — только при необходимости
Оптимизация RAG
Лучший поиск, ранжирование, минимизация контекста
Преиспользование рекомендаций
Сохранение ответов и контента в RAG для повторного использования
Вывод
Прибыльность AI-продукта напрямую зависит от контроля вычислительных затрат и эффективности использования моделей.
