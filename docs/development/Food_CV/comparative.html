📊 t-статистика: сравнительный анализ моделей

Методика t-статистики

Для каждой модели рассчитывается одновыборочный t-критерий (двусторонний):

t = (x̄ − μ) / (s / √n)

  • x̄ — среднее предсказание модели по n блюдам
  • μ — эталонное значение (из food_catalog.json)
  • s — стандартное отклонение ошибок предсказания
  • n — количество блюд с непустыми данными для данной модели

Гипотеза H₀: средняя ошибка модели не отличается от нуля (модель несмещена).

При |t| > 2.074 (p < 0.05, df≈22) гипотеза отвергается — модель имеет статистически значимое смещение.

Выводы по применимости для дневника питания

  • Простые блюда (завтраки, монопродукты): Все модели показывают MAE < 15% — пригодны для автоматического ввода.

🔬 Методика проверки достоверности распознавания блюд

1. Формирование эталонного датасета

Для каждого из 23 блюд определён эталонный состав из food_catalog.json — верифицированный нутрициологом набор показателей: калорийность (ккал), белки, жиры, углеводы (г), а также индекс нутритивной плотности NDI (0–10). Каждое блюдо имеет фотографию высокого разрешения (200×200 px), сохранённую в локальном каталоге images/.

2. Процедура тестирования CV-моделей

Каждая модель получает на вход изображение блюда и возвращает:

  • Состав ингредиентов — список распознанных продуктов
  • КБЖУ — калории, белки, жиры, углеводы (в абсолютных значениях)
  • Оценку качества — текстовое описание, плюсы/минусы/рекомендации
  • Уверенность — внутреннюю оценку модели (0–100%)

3. Метрики точности

  • Совпадение ингредиентов (%) — доля ингредиентов из эталона, распознанных моделью. Учитывается семантическая близость (например, «говядина» ≈ «мясо»).
  • MAE (Mean Absolute Error, %) — средняя абсолютная ошибка предсказания относительно эталона: MAE = (|pred − actual| / actual) × 100% для каждого макронутриента.
  • Дисперсия ошибки — разброс ошибок предсказания, характеризующий стабильность модели на разных типах блюд.

4. Статистическая проверка (t-критерий)

Для каждой модели и каждого показателя рассчитывается одновыборочный t-критерий (двусторонний, α = 0.05):

t = (x̄ − μ) / (s / √n)

где x̄ — среднее предсказание модели по n блюдам, μ — эталонное значение, s — стандартное отклонение ошибок. Нулевая гипотеза H₀: модель не имеет систематического смещения (средняя ошибка = 0).

Критическое значение |t| > 2.074 (df = 22, p < 0.05). Если |t| превышает порог — модель статистически значимо завышает или занижает показатель.

5. Ограничения методики

  • Данные собраны из реальных прогонов CV-моделей (tune_model/*.json), но не все модели проанализировали все 23 блюда
  • Оценка совпадения ингредиентов — экспертная, а не автоматическая
  • Размер датасета (23 блюда) достаточен для первичного сравнения, но не для финальных выводов
  • Фотографии блюд имеют разное освещение и ракурс, что влияет на качество распознавания
