# Визуализация pipeline поиска знаний — варианты

> Цель: отобразить цепочку поиска (Источники → LLM → Валидация → БЗ) с текущим статусом каждого шага. Встроить в существующий React-админку без перестройки архитектуры.

---

## 1. React Flow (@xyflow/react) — интерактивный DAG

**GitHub:** xyflow/xyflow — 37k★  
**npm:** `@xyflow/react` v12.10, MIT  
**Bundle:** ~58 KB gzipped  
**Интеграция:** Нативный React-компонент, `<ReactFlow nodes={...} edges={...} />`

### Возможности
- Кастомные React-ноды для каждого шага пайплайна (с иконками, статусами, прогресс-барами)
- Анимированные края (running = пунктир, completed = сплошной, failed = красный)
- Auto-layout через `dagre` — позиции нод рассчитываются автоматически
- Зум/панорамирование/миникарта из коробки
- Можно сделать drag-n-drop для ручного конфигурирования пайплайна

### Пример ноды
```tsx
function PipelineNode({ data }) {
  const statusColor = { running: '#ffd43b', completed: '#4ecdc4', failed: '#ff6b6b', idle: '#e8e8f0' };
  return (
    <div style={{ border: `2px solid ${statusColor[data.status]}`, borderRadius: 8, padding: 8, background: '#fff', minWidth: 140 }}>
      <div style={{ fontSize: 11, color: '#999' }}>{data.step}</div>
      <div style={{ fontWeight: 600, fontSize: 13 }}>{data.label}</div>
      {data.meta && <div style={{ fontSize: 10, color: '#888', marginTop: 2 }}>{data.meta}</div>}
    </div>
  );
}
```

### Когда подходит
- Нужен полноценный визуальный редактор pipeline (можно перетаскивать ноды, менять порядок)
- Пользователям нужно видеть топологию процесса
- Планируется 10+ различных шагов с ветвлениями

## 2. Step-tracker (Stepped Timeline) — прогресс выполнения

**Суть:** Горизонтальная последовательность шагов с индикатором статуса (как GitHub Actions или CI-пайплайн).

**Библиотек не нужно** — это 50 строк CSS + React.

```
[ PubMed ] ──→ [ LLM Summarize ] ──→ [ Expert Review ] ──→ [ Knowledge Base ]
   ✓                ✓                     ⏳                      —
 125 found        3 extracted            waiting                 pending
```

### Реализация
```tsx
const STEPS = [
  { id: 'search', label: 'Поиск', icon: '🔍', status: 'done', meta: '125 found' },
  { id: 'llm', label: 'LLM', icon: '🤖', status: 'done', meta: '3 extracted' },
  { id: 'review', label: 'Review', icon: '👤', status: 'running', meta: '2 of 3' },
  { id: 'store', label: 'БЗ', icon: '💾', status: 'idle', meta: '' },
];
// Рендер: flex-row, каждый шаг — кружок + линия + метка
```

### Когда подходит
- Линейный пайплайн без ветвлений (как у нас: поиск → LLM → валидация → БЗ)
- Нужно компактно показать статус текущей сессии
- Минимальный bundle (zero зависимостей)

## 3. Timeline-лог (как Chrono UI) — история выполнения сессии

**Суть:** Вертикальная хронология событий внутри сессии поиска.

```
09:32:15  ▶ Запущен поиск PubMed...               125 найдено
09:32:47  ▶ LLM-анализ: summarization...          3 статьи
09:33:20  ▶ Ожидание валидации экспертом...        2 ожидают
09:34:00  ✓ Эксперт одобрил: "Vitamin D protocol"  #42
09:34:01  ✓ Статья добавлена в БЗ                 total: 357
```

### Когда подходит
- Нужна детальная хронология каждого запуска
- Пользователи хотят видеть лог в реальном времени
- Комбинируется с Step-tracker (сверху статус, снизу лог)

## 4. Dashboard-панель (текущая реализация)
