## Алгоритм работы (data flow)

```
Вход: DT (фенотип, TTM, барьеры, опционально генетика/метилирование)
      + выбранный протокол P + цели G

1. Candidates = filter_catalog(P, G)          — отбор из каталога
2. for each INT_i:
     w_i = α₁·evidence + α₂·goal_alignment + α₃·twin_comp + α₄·regress + α₅·barrier
     f_i = base_intensity × readiness_modifier(TTM.stage)
     r_i = default_regularity × schedule_adjust(DT.preferences.time_slots)
3. S = sort_by_importance(Candidates)
4. S = select_top_k(S, K=min(12, max=12))
5. S = apply_constraints(S, DT.constraints)   — противопоказания, burden, баланс

Пользователь корректирует:
  - Меняет f_i через ползунок → локальное обновление
  - Удаляет INT_i → пересчёт w_j оставшихся (нормализация)
  - Добавляет INT_j → w_j рассчитывается по формуле выше
  - «Вернуть стандарт» → сброс к алгоритмическому набору

При сохранении:
  - План фиксируется в PlansProvider
  - Запускается 7-дневный цикл пересчёта (см. 5.5 динамика)
  - Прогноз Z = β₀ + Σβ_i·X_i (adherence/weight delta)
```

## Приоритет реализации

1. **MVP** — Экран 2 (обзор с ползунками) + Экран 5 (сохранение). Без генетики, w_i по 3 факторам (evidence, goal, twin), f_i = 1.0 для всех.
2. **V2** — Добавить Экран 3 (добавление) + Экран 4 (детали). TTM-адаптация f_i.
3. **V3** — Регрессионная модель + генетика/метилирование. Dynamic recalculation.
