# Healora Application - Screens and Views

This document lists all distinct screens, views, and modals in the Healora application based on code analysis.

## Main Application Screens (Routed Pages)

These are the primary screens accessible via navigation routes:

### 1. Digital Twin Dashboard
- **Path:** `/` or `/digital-twin`
- **Component:** `DigitalTwin.jsx`
- **Description:** Main dashboard showing patient's digital twin with:
  - 6-section attribute catalog (Demographics, Vitals, Labs, Lifestyle, Genetics, Medical)
  - Inline-editable health parameters with history tracking
  - Timeline view with intervention scheduling
  - Plan prescription interface
  - Goal setting chat
  - Food selector and voice input
  - Parameter history and assessment tools
- **Key Features:** Drag-and-drop intervention planning, real-time simulation, comprehensive health tracking

### 2. AI Chat Interface
- **Path:** `/chat`
- **Component:** `ChatInterface.jsx` (wrapped in `PhoneContainer`)
- **Description:** Conversational interface with HEALORA AI assistant featuring:
  - Message exchange with GigaChat/OpenAI providers
  - Task completion system with star rewards
  - Quick action buttons for common queries
  - Knowledge quizzes for learning
  - Sidebar for data sources and action log
- **Key Features:** Multi-turn conversations, task-based learning, contextual health advice

### 3. Health Journey Path
- **Path:** `/path`
- **Component:** `ProgressPath.jsx` (wrapped in `PhoneContainer`)
- **Description:** Visualization of patient's health progress showing:
  - Profile selection from test cases
  - Progress statistics (days, phase, completion %)
  - Recommended interventions grid
  - Device data import zone (wearables, bio-data)
  - Digital twin state indicators (Energy, Focus, Stress, Sleep, Mood)
- **Key Features:** Multi-profile comparison, device integration, state tracking

### 4. Goals Tracking
- **Path:** `/goals`
- **Component:** `Goals.jsx` (wrapped in `PhoneContainer`)
- **Description:** Health goal management interface displaying:
  - Current goals with progress bars (water, sleep, steps, meditation, reading)
  - Category summary grid (Knowledge, Activity, Recovery, Nutrition, Specialists)
- **Key Features:** Goal progress visualization, category-based tracking

### 5. Nutrition Diary
- **Path:** `/diary`
- **Component:** `NutritionDiary.jsx` (wrapped in `PhoneContainer`)
- **Description:** Mobile-optimized food and symptom tracking with:
  - Daily navigation (Day 0-30)
  - Four meal entries (Breakfast, Lunch, Dinner, Snack)
  - Photo upload, time tracking, KBZU nutritional data
  - Water intake slider and 5-mood tracking system
  - Voice recording for meal descriptions
  - Audio file upload for doctor consultations
- **Key Features:** Multimedia meal logging, voice input, comprehensive symptom tracking

### 6. Protocol Selection Flow
- **Path:** `/flow`
- **Component:** `DecisionFlowSimulation.jsx` (wrapped in `PhoneContainer`)
- **Description:** Interactive protocol recommendation wizard with:
  - Multi-phase chat flow (Welcome → Current stats → Top-3 protocols → Data request → Selection → Completion)
  - Personalized protocol scoring based on patient data
  - Missing data identification with effort/impact analysis
  - System log showing calculation details
- **Key Features:** AI-driven protocol recommendation, data gap analysis, personalized scoring

## Persistent UI Panels

These components remain visible across multiple screens:

### 7. User Profile Selector
- **Component:** `UserAvatarPanel.jsx`
- **Location:** Left sidebar
- **Description:** Profile management interface showing:
  - List of patient profiles with avatars and risk indicators
  - Status dots (green/yellow/red for risk levels)
  - Basic profile info (name, age, ID, vitals)
  - Stars and achievement tracking
  - Photo selector for profile customization
- **Key Features:** One-click profile switching, visual risk assessment, photo management

### 8. Interventions & Protocols Panel
- **Component:** `InterventionsPanel.jsx`
- **Location:** Right sidebar
- **Description:** Dual-tab interface for managing health interventions:
  - **Interventions Tab:**
    - Categorized intervention catalog (Sleep, Physical, Mental, Food, Medical, Supplements)
    - Filter buttons and subgroup views (Vitamins, Minerals, Diets, Food Habits)
    - Impact/Evidence/Periodicity columns with add-to-cart functionality
    - Detail popups for interventions and supplements
    - Legend explaining I/E/Per abbreviations
  - **Protocols Tab:**
    - 17 expandable protocol cards (Glycemic Control, Circadian Eating, Rapid Weight Loss, etc.)
    - Protocol details (goals, interventions, red flags, recommendations)
    - Development backlog viewer
- **Key Features:** Drag-and-drop intervention planning, protocol browsing, cart-based ordering

### 9. Data Sources Footer
- **Component:** `SourcesFooter.jsx`
- **Location:** Collapsible bottom panel
- **Description:** Health data integration interface featuring:
  - Source activation status counter
  - Drag-and-drop source cards (Wearables, Voice, Medical, Food, Genetics, Mental)
  - Active source tags display
  - Profile configuration drop zone
  - Navigation to Digital Twin dashboard
- **Key Features:** Visual source management, drag-and-drop integration, one-click dashboard access

## Embedded View Components

These components are reused across different contexts:

### 10. Profile View
- **Component:** `ProfileView.jsx`
- **Usage:** Used in Digital Twin, Phone Simulator, and popups
- **Description:** Patient profile display with:
  - Avatar with risk-colored mood indicator
  - Core demographics (name, sex, age, level, risk)
  - Six-section navigation (Overview + Demographics, Vitals, Labs, Lifestyle, Genetics, Medical)
  - Stat cards for key metrics (Stars, BMI, BP, Glucose, Sleep, Risk)
  - Healora Score progress visualization
  - Detailed attribute views with current/target/norm values
- **Key Features:** Section-based organization, visual risk indicators, metric tracking

### 11. Plan Prescription View
- **Component:** `PlanView.jsx`
- **Usage:** Used in Digital Twin, Phone Simulator, and prescription popups
- **Description:** Health plan visualization showing:
  - Header with plan template selector and PDF export
  - Doctor-patient-info block with date and progress bar
  - Target parameters section with start/current/target charts
  - RPP (eating disorder) warning block when applicable
  - Intervention table with schedule, description, codes, and completion tracking
  - Doctor notes section and plan status controls (Active/Stopped/Archived)
  - QR code and app mockup (in full mode)
- **Key Features:** Plan customization, progress tracking, status management, export capability

### 12. Diary View
- **Component:** `DiaryView.jsx`
- **Usage:** Used in Nutrition Diary screen and popups
- **Description:** Nutrition and symptom logging interface with:
  - Day-specific header ("Nutrition Diary -- Day X")
  - Four meal sections with photo upload and food catalog
  - Nutritional inputs (calories, protein, fat, carbs)
  - Description fields with voice input capability
  - Time and duration tracking per meal
  - Water intake slider
  - Five-mood tracking system (Energy, Mood, Sleep, Stress, Digestion)
  - RPP (eating disorder) section with type/frequency tracking
  - Voice recording with transcript display
  - Save submission button
- **Key Features:** Multimedia meal logging, voice input, comprehensive symptom tracking, RPP support

## Popups and Modals

These are overlay interfaces triggered by specific actions:

### 13. Voice Input Popup
- **Trigger:** Mic button in section headers or meal descriptions
- **Content:** Web Speech API interface with:
  - 8-language selector (Russian, English, etc.)
  - Microphone device picker with level meter
  - Real-time transcript display
  - Per-field recording buttons for targeted input
  - Parsed value display and confirmation
- **Usage:** Voice-controlled parameter entry and meal description

### 14. Food Selector Popup
- **Trigger:** Food catalog button in meal planners
- **Content:** Searchable food database with:
  - 1000+ food items with nutritional data
  - Keyword search and filtering
  - Category browsing
  - Selection confirmation and addition to current meal
- **Usage:** Meal planning and nutritional tracking

### 15. Plan Prescription Popup
- **Trigger:** Plan button in profile header or plan edit action
- **Content:** Full HEORA prescription layout showing:
  - Patient and doctor information
  - Current date and plan validity period
  - Intervention schedule with timing and frequency
  - Target health parameters with projected trajectories
  - Doctor notes section
  - Plan status controls and action buttons
  - QR code for mobile access
- **Usage:** Plan review, editing, status management, and export

### 16. Intervention/Supplement Detail Popup
- **Trigger:** Clicking on intervention/supplement name in panels or tables
- **Content:** Detailed information including:
  - Code, name, and description
  - Category classification
  - Impact score (1-10) and evidence level (A-D)
  - Periodicity (daily, weekly, monthly, etc.)
  - Linked protocols (for interventions)
  - Source and type information (for supplements)
- **Usage:** Detailed reference before adding to plan

### 17. Legend Popup
- **Trigger:** "?" button in interventions panel
- **Content:** Explanation of coding system:
  - **I (Impact):** 1-10 scale measuring effect magnitude
  - **E (Evidence):** A-D scale (A=highest, D=lowest) for scientific backing
  - **Per (Periodicity):** Frequency patterns (D=daily, W=weekly, M=monthly, Y=yearly, P=as needed)
- **Usage:** Reference for understanding intervention ratings

### 18. Development Backlog Modal
- **Trigger:** Version link in interventions panel footer
- **Content:** Structured view of project backlog showing:
  - Version information and release notes
  - Completed items categorized by type
  - Todo items grouped by feature area
  - In-progress work and blockers
- **Usage:** Development tracking and transparency

### 19. Parameter History Popup
- **Trigger:** History icon on parameter rows
- **Content:** 7-day visualization showing:
  - Weekday column layout (Mon-Sun)
  - Current value highlighting
  - Trend indicators and variation analysis
  - Target comparison and deviation alerts
- **Usage:** Short-term parameter trend analysis

### 20. Task Tracking Popup
- **Trigger:** Tasks badge in plan prescription
- **Content:** Intervention completion tracking showing:
  - List of unique intervention codes in plan
  - Scheduled vs. completed counts for each
  - Percentage completion metrics
  - Color-coded status indicators
- **Usage:** Adherence monitoring and gap identification

### 21. Assessment Popup
- **Trigger:** Assessment button in dashboard
- **Content:** Health evaluation form with:
  - Standardized health questionnaires
  - Symptom severity scales
  - Risk factor assessments
  - Quality of life measurements
- **Usage:** Baseline assessment and progress evaluation

### 22. Diet Preferences Popup
- **Trigger:** Diet preferences button in nutrition planning
- **Content:** Toggle interface for:
  - Common dietary patterns (vegetarian, vegan, paleo, keto, etc.)
  - Religious restrictions (halal, kosher, etc.)
  - Custom preference text entry
  - Allergy and intolerance specification
- **Usage:** Personalizing nutritional recommendations

### 23. Practice Information Popup
- **Trigger:** Practice badge in medical section
- **Content:** Traditional medical practice details from:
  - Text files in src/assets/data/practices/
  - Historical and cultural context
  - Modern applicability and contraindications
  - Integration notes with conventional medicine
- **Usage:** Educational reference for holistic approaches

### 24. Prototype Screen Lightbox
- **Trigger:** Prototype reference in development tools
- **Content:** Gallery of reference screens showing:
  - 10 prototype screen designs
  - Annotations and descriptions
  - Usage context and user flow notes
  - Design rationale and alternatives considered
- **Usage:** Design reference and development planning

### 25. Authentication Popup
- **Trigger:** Login button in top app bar
- **Content:** Account access interface with:
  - Email/password login form
  - Registration option (email/password)
  - Tab switching between modes
  - Validation and error handling
  - Remember me and forgot password links
- **Usage:** Secure account access and registration

## Special Simulation Views

### 26. Phone Simulator Views
- **Component:** `PhoneSimulator.jsx`
- **Description:** Legacy simulation mode showing:
  - Tabbed interface (Profile/Diary/Plan/Chat)
  - Embedded ProfileView, DiaryView, PlanView components
  - 60-day simulation history with adherence metrics
  - Day-by-day notes and chat transcripts
  - Z-score tracking and visualization
- **Usage:** Historical data review and simulation analysis
