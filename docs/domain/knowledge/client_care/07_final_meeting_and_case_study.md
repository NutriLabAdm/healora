## Структура итоговой встречи

```
АНАЛИЗ РАБОТЫ → ПОДВЕДЕНИЕ ИТОГОВ → ДОГОВОР НА ДАЛЬНЕЙШЕЕ СОТРУДНИЧЕСТВО → ОТЗЫВ
```

---

## 1. Анализ работы

Задача этапа — **увидеть результат и показать его клиенту**. Клиент быстро привыкает к хорошему и может забыть, что было «до».

### 1.1. Точка А — с чего начинали

Вспомнить и зафиксировать исходное состояние:
- Жалобы и симптомы клиента на старте
- Запрос и постановка цели (SMART)
- Результаты анкетирования и первичной консультации
- Фото/анализы/замеры «до»

> *«К хорошему привыкаешь быстро. Клиент может просто забыть, что было месяц назад.»*

### 1.2. Что было сделано — разбор действий

| Вопрос | Назначение |
|---|---|
| Какие действия принесли результат? | Закрепить успешные практики |
| Какие действия НЕ принесли результата? | Отфильтровать неэффективные методы |
| Какие рекомендации НЕ были выполнены и почему? | Понять барьеры (среда, мотивация, сложность) |
| Что можно улучшить? | Рефлексия для будущей работы |

Даже самые маленькие шаги важно отметить. Если клиент просто начал пить воду — это результат.

### 1.3. Точка В — текущий результат

- Где мы сейчас?
- Что изменилось в симптомах и жалобах?
- Показать динамику: таблица, треклист, график изменений

> *«Расписать лучше всего прямо — показать клиенту, какого результата мы достигли.»*

## 2. Подведение итогов

### 2.1. Ключевые вопросы

| Вопрос | Комментарий |
|---|---|
| Достигли ли мы цели? | Сравнить с минимальной (100% выполнимой) и максимальной (вау-цель) |
| Довольны ли вы результатом? | Если да — отлично. Если нет — выяснить причину |
| Что можно улучшить? | Только после честного признания результатов |

### 2.2. Глубинные вопросы

| Вопрос | Назначение |
|---|---|
| Какой результат стал для вас самым значимым? | Выявить ценность для клиента |
| Какую рекомендацию было сложнее всего выполнить? | Понять барьеры для следующего этапа |
| Какая практика стала вашей любимой? | Закрепить то, что работает |
| Какая новая привычка вам больше всего нравится? | Создать позитивное якорение |

### 2.3. Работа с недовольством

Если клиент недоволен результатом:
- Почему? Что можно улучшить?
- Недостаточно времени?
- Недостаточно ресурсов у клиента?
- Проблема требует долгосрочной работы?

## 3. Договор на дальнейшее сотрудничество

Итоговая встреча — не конец, а **возможность начать новый этап**.

### 3.1. Переход к продолжению работы

| Вопрос | Комментарий |
|---|---|
| Есть ли желание улучшить свои результаты? | Мягкий переход от точки В к точке С |
| Какая ваша следующая цель? | Клиент формулирует сам |
| Нужна ли поддержка или справитесь самостоятельно? | Оценка готовности к самостоятельному ведению |
| Готовы ли работать дальше? | Прямой вопрос о продолжении |

### 3.2. Точка С — следующая цель

Обсудить и зафиксировать:
- Новый запрос клиента (коррекция веса → спортпит, ЖКТ → энергия)
- Изменение формата работы (еженедельно → раз в 2 недели)
- Новые сроки и оплата

---

## 4. Отзыв клиента

Запросить отзыв **в конце итоговой встречи**, пока эмоции свежи.

| Формат отзыва | Когда использовать |
|---|---|
| Текст | Для постов и портфолио |
| Аудиосообщение | Для подкастов и каналов |
| Видео | Наиболее продающий формат |

**Важно:** отзыв не просто реклама — это подтверждение клиенту, что работа была проделана не зря.

## 5. Оформление кейса

Кейс — лучшая реклама нутрициолога. Демонстрация результата работает сильнее любых слов об экспертности.

### 5.1. Зачем нужен кейс

| Функция | Описание |
|---|---|
| **Обозначение проблем** | Показать, с какими запросами можно работать (не только вес — ЖКТ, сон, гипотиреоз, энергия) |
| **Проработка страхов** | «Если этот человек смог — я тоже смогу». Снимает возражения: нет времени / дорого / не получится |
| **Демонстрация экспертности** | Разбор нюансов, индивидуальных особенностей, сложностей и их решений |
| **Мотивация** | Результат «до/после» вдохновляет новых клиентов |

### 5.2. Форматы кейса

| Формат | Описание |
|---|---|
| **Пост в соцсетях** | Развёрнутая история с точкой А → процесс → точка В |
| **Сторис / рилс** | Серия публикаций: ежедневные отчёты, эмоции, результаты по неделям |
| **Портфолио (PDF / презентация)** | Сборник кейсов для рассылки потенциальным клиентам |
| **Реалити / сериал** | Ведение клиента в прямом эфире: подписчики вовлечены, ждут продолжения |

### 5.3. Структура кейса

| Элемент | Описание |
|---|---|
| **Точка А** | Анамнез, жалобы, запрос, цель клиента |
| **Точка В** | Результат, что изменилось, достигли ли цели |
| **Фото ДО / ПОСЛЕ** | Обязательно — визуально, не подделать |
| **Отзыв клиента** | Эмоциональный, с акцентами на главное |

### 5.4. Правила оформления

| Правило | Описание |
|---|---|
| **Просто, понятно, эмоционально** | Кейс должен вызывать чувства, а не только информировать |
| **Яркие акценты** | Ключевые слова выделены цветом/шрифтом — сразу бросаются в глаза |
| **Цепляющие картинки** | Визуальное восприятие первично: фото, инфографика, скрины отчётов |
| **Узнаваемость** | Единый стиль, хэштеги (#ИмяКлиента), персонаж-эмодзи, регулярность |
| **Показывать трудности** | Не бояться показывать сложные моменты — это повышает доверие |

## 6. Пример из практики (Ксения Конопля)

**Цель кейса:** прорекламировать новую индивидуальную программу.

**Клиент:** ответственная девушка, дала разрешение на публикацию.

**Формат:** сторис ежедневно + пост раз в неделю с эмоциями.

**Узнаваемость:** #ЛАДим_Настю, персонаж-эмодзи, единое оформление, регулярность.

**Акценты:**
- Описание программы, задачи
- Индивидуальный подход
- Трудности → решения (скрины отчётов клиентки с грустью, усталостью — и поддержка специалиста)
- Результат → эмоции
- Отзывы с подсветкой ключевых фраз

**Структура сторис:**
1. Аватар клиентки (узнаваемый)
2. Проблемы и боль → цели-мечты
3. Направления работы (питание, движение, сон, самоощущение)
4. Ежедневные задачи по 1–2 направлениям
5. Трудности (показать отчёты клиента)
6. Сравнение «до/после» (клиентка сама замечает разницу)
7. Итоговый пост с отзывом и акцентами

## 7. Алгоритм действий

```
┌─────────────────────────────────────────────────────────────┐
│ 1. ЗА НЕДЕЛЮ ДО                                             │
│    Напомнить клиенту о завершении, попросить подготовиться  │
├─────────────────────────────────────────────────────────────┤
│ 2. ИТОГОВАЯ ВСТРЕЧА                                         │
│    Точка А → анализ → точка В → цель достигнута? →          │
│    → будущее? → отзыв                                       │
├─────────────────────────────────────────────────────────────┤
│ 3. ПОСЛЕ ВСТРЕЧИ                                            │
│    Получить отзыв → запросить разрешение на кейс →          │
│    → собрать материалы (фото, скрины, таблицы)              │
├─────────────────────────────────────────────────────────────┤
│ 4. ОФОРМЛЕНИЕ КЕЙСА                                         │
│    Выбрать формат (пост/сторис/портфолио) →                 │
│    → структура (А + В + фото + отзыв) → оформить →          │
│    → опубликовать с хэштегом                                │
├─────────────────────────────────────────────────────────────┤
│ 5. ДАЛЬНЕЙШЕЕ СОТРУДНИЧЕСТВО                                │
│    Если клиент продолжает — новый контракт, точка С.        │
│    Если нет — остаётся в базе для реферальных программ.     │
└─────────────────────────────────────────────────────────────┘
```

## Промпты для презентации

### Для клиента
> **Заголовок:** «Без названия: план действий для улучшения здоровья»
>
> **Теория:**
> - Цель протокола: улучшение показателей здоровья
> - Кому подходит и когда начинать
> - Основные шаги и их последовательность
> - Ожидаемые результаты и сроки
>
> **Практика:**
> - Пошаговый план на неделю/месяц
> - Чек-лист ежедневных действий
> - Трекер прогресса (что записывать)
> - Когда ждать первых результатов
>
> **Дисклеймер:** Протокол носит ознакомительный характер. Индивидуальная программа составляется специалистом.

### Для специалиста
> **Заголовок:** «Без названия: клинический протокол ведения»
>
> **Теория:**
> - Обоснование протокола (биологические механизмы)
> - Показания и противопоказания (стратификация пациентов)
> - Доказательная база и уровни рекомендаций
>
> **Практика:**
> - Алгоритм: скрининг → стратификация → интервенция → мониторинг
> - 
> - Критерии эффективности и точки коррекции
> - Протокол безопасности (побочные эффекты, лабораторный контроль)
>
> **Дисклеймер:** Референсные протоколы. Адаптируются под конкретного пациента.
