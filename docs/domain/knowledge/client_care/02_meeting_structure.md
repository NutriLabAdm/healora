## 4-шаговая структура встречи

Этот план повторяется на **каждой** встрече с клиентом (еженедельной или раз в 2 недели).

```
СИНХРОНИЗАЦИЯ → ПРЕДЫДУЩИЙ ОПЫТ → ВЫДАЧА РЕКОМЕНДАЦИЙ → ПЛАН РАБОТЫ
```

---

### Шаг 1. Синхронизация

Задача: установить контакт, снять тревогу, настроить на продуктивный разговор.

| Действие | Примеры вопросов |
|---|---|
| **Установить контакт** | «Как дела? Как настроение?» |
| **Похвалить** | «Отлично выглядите! Какие у вас красивые серьги!» |
| **Спросить об изменениях** | «Как успехи? Что получилось внедрить?» |
| **Выявить сложности** | «Были ли сложности при выполнении задания? Какие?» |
| **Оценить готовность** | «Готовы ли вы добавить нагрузку?» |

Важно: начинаем с позитива. Не «что не получилось», а «что удалось». Клиент не должен чувствовать себя провинившимся учеником.

---

### Шаг 2. Предыдущий опыт клиента

Перед тем как дать новую рекомендацию, выясняем, что клиент **уже знает и пробовал** по этой теме.

| Что выясняем | Примеры вопросов |
|---|---|
| **Что уже знает/пробовал** | «Что вы уже знаете по этой теме? Пробовали ли раньше худеть / менять питание / заниматься спортом? Расскажите о своём опыте» |
| **Неизменяемые установки** | «Что вы не будете менять в своей жизни ни при каких обстоятельствах?» |
| **С чего проще начать** | «Как ты думаешь, с чего тебе будет проще начать?» — клиент сам выбирает первый шаг |
| **Ожидания** | Какой результат клиент считает приемлемым |

Почему это важно:
- Не тратить время на то, что клиент уже знает
- Не вступать в конфликт с установками клиента (вегетарианство, любовь к жареной картошке, религия)
- Включить клиента в процесс: он выбирает шаг, потому что лучше всех знает свою жизнь

---

### Шаг 3. Выдача рекомендаций

Детально описано в `client_care/03_recommendation_delivery.md`.

Кратко: рекомендации должны быть простыми, с примерами и метафорами, с вовлечением клиента и совместным поиском решений.

---

### Шаг 4. План работы для клиента

| Элемент плана | Описание |
|---|---|
| **Минимум пунктов** | Легко запомнить, выполнить, проконтролировать. 1–3 пункта на неделю |
| **Формат отчёта** | Договориться, как клиент показывает выполнение: фото тарелок, скрин фитнес-приложения, галочка в таблице-трекере |
| **Отработка возражений** | «Что может помешать?» — проиграть сложные ситуации заранее |

## Типичные ошибки на встречах

| Ошибка | Последствие | Решение |
|---|---|---|
| Начать сразу с рекомендаций без синхронизации | Клиент не вовлечён, чувствует себя «подопытным» | Всегда начинать с «Как дела?» |
| Пропустить шаг «Предыдущий опыт» | Рекомендация уже пробована и не сработала | Спрашивать перед каждой новой темой |
| Слишком много пунктов в плане | Ничего не будет выполнено | 1–3 пункта, не больше |
| Не оговорили формат отчёта | Клиент не знает, как и когда отчитываться | Чётко: «каждый вечер фото тарелки в WhatsApp» |
