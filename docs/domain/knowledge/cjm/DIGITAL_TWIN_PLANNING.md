## Пользовательский путь: Планирование интервенций в Digital Twin

### Flow: Профиль → Риски → Цели → Протоколы → Таймлайн → Корректировка

#### Шаг 1: Профиль (входная точка)

```
Пользователь открывает Digital Twin для своего профиля
↓
Система показывает:
  - Текущие биомаркеры и параметры здоровья
  - Звезды / прогресс
  - Оценка здоровья (риск-скор)
```

**UI Elements:**
- Карточка профиля с аватаром и ключевыми метриками
- Индикатор здоровья (цветовая шкала)
- Кнопка "Начать планирование"

#### Шаг 2: Риски и возможные цели

```
Система анализирует профиль и показывает:
  - Выявленные риски (алерты по биомаркерам)
  - Рекомендуемые цели (на основе отклонений от нормы)
  - Потенциал улучшения
```

**UI Elements:**
- Секция "Риски" — красные/оранжевые алерты
- Секция "Возможные цели" — рекомендации системы
- Пользователь выбирает одну или несколько целей

**Примеры целей:**
- Снизить глюкозу до нормы
- Улучшить качество сна
- Снизить стресс
- Повысить VO2 max

#### Шаг 3: Выбор протоколов

```
Пользователь выбирает цели
↓
Система предлагает релевантные протоколы для выбранных целей
↓
Пользователь назначает один или несколько протоколов
```

**UI Elements:**
- Карточки протоколов с описанием и количеством интервенций
- Фильтрация по категориям (питание, сон, физическое, медицинское)
- Кнопка "Добавить протокол"
- Показ совместимости протоколов

**Протоколы:**
| ID | Название | Цель | Интервенции |
|----|----------|------|-------------|
| 1.1 | Гликемический контроль | Снижение глюкозы | 04_1, 04_2, 04_3, 04_6, 02_3, 03_5 |

| 2.1 | Метаболические риски | Снижение метаболического риска | 02_1, 02_2, 04_1, 07_2 |
| 2.2 | Гормональные нарушения | Баланс гормонов | 05_5, 03_1, 03_6 |

#### Шаг 4: Планирование на таймлайне

```
Назначенные протоколы автоматически распределяются на 90 дней
↓
Интервенции размещаются согласно регулярности:
  - daily → каждый день (0, 1, 2, ... 90)
  - weekly → каждые 7 дней (0, 7, 14, ... 84)
  - yearly → один раз (в начале)
↓
Пользователь видит план на таймлайне
```

**UI Elements:**
- Таймлайн 90 дней с протоколами как группами
- Каждая группа сворачиваемая (▼/▶)
- Интервенции визуализированы как точки на треках
- Playhead для текущего дня симуляции

**Поведение по тарифам:**

| Тариф | Планирование | Ручная правка | AI |
|-------|--------------|---------------|-----|
| Free | Протокол → автоплан | Нет | Нет |
| Basic+ | Цели → AI → план | Да (убрать, передвинуть, добавить) | Да |
| Premium | Цели + история → AI → план | Да + экспертная оценка | Да + эксперт |

#### Шаг 5: Корректировка плана

```
Пользователь может:
  - Убрать интервенцию (клик на ×)
  - Передвинуть на другой день (drag & drop)
  - Добавить интервенцию из каталога
  - Убрать весь протокол
  - Добавить еще протокол
```

**UI Elements:**
- Клик на интервенцию → попап с деталями + кнопка "Убрать"
- Drag & drop на таймлайне
- Кнопка "+ Интервенция" открывает каталог
- Кнопка "+ Протокол" открывает выбор протоколов

## Тарифные модели и возможности

### Free (Фримиум)

```
Доступно:
  - Просмотр профиля и рисков
  - Выбор целей из рекомендованных
  - Добавление одного базового протокола
  - Автоматическое планирование на 90 дней
  - Просмотр таймлайна
  - Симуляция (play/stop)

Ограничено:
  - Только базовые протоколы (1.1, 1.3, 1.5)
  - Нет ручной корректировки
  - Нет AI-планировщика
  - Нет дополнительных интервенций
```

### Basic+ (базовый+)

```
Доступно:
  - Все из Free
  - Все протоколы
  - AI-планировщик (подбор под цели)
  - Ручная корректировка:
    - Убирать интервенции
    - Передвигать на другие дни
    - Добавлять из каталога
  - Несколько протоколов одновременно
  - Сохранение плана

Ограничено:
  - Нет экспертной оценки
  - Нет анализа истории
```

### Premium (премиум)

```
Доступно:
  - Все из Basic+
  - AI с учетом истории пользователя
  - Экспертная оценка плана
  - Рекомендации по оптимизации
  - Интеграция с внешними данными
  - Приоритизация интервенций
  - Прогноз результатов

Особенности:
  - AI анализирует историю и предлагает персонализированный план
  - Эксперт (врач/нутрициолог) проверяет план
  - Система предлагает корректировки на основе данных
```

## AI-планировщик (Basic+ и Premium)

### Входные данные AI:

```
Для Basic+:
  - Выбранные цели
  - Текущие биомаркеры
  - Выбранные протоколы

Для Premium:
  - Все из Basic+
  - История интервенций и результатов
  - Генетические данные (если есть)
  - Данные с носиемых устройств
  - Предыдущие планы и их эффективность
```

### Выход AI:

```
Для Basic+:
  - Предложенный план интервенций
  - Обоснование выбора
  - Ожидаемый результат

Для Premium:
  - Оптимизированный план с учетом истории
  - Оценка рисков и противопоказаний
  - Приоритеты (что делать в первую очередь)
  - Прогноз улучшений по биомаркерам
  - Рекомендации эксперта
```

## Состояния интервенций на таймлайне

| Состояние | Цвет | Описание |
|-----------|------|----------|
| Запланировано | Серый | Интервенция добавлена, день не наступил |
| Активно | Цвет категории | День интервенции = текущий день |
| Выполнено | Зеленый | Симуляция прошла день, интервенция сработала |
| Пропущено | Красный | День прошел, интервенция не выполнена |
| Убрано | — | Пользователь удалил интервенцию |

## Переходы между экранами

```
Профиль
    ↓
Риски и цели → Выбор целей
    ↓
Протоколы → Выбор одного или нескольких протоколов
    ↓
Планирование (AI для Basic+/Premium)
    ↓
Таймлайн (автоплан или AI-план)
    ↓
Корректировка (ручная правка для Basic+/Premium)
    ↓
Симуляция
    ↓
Результаты → Возврат к целям или новый цикл
```
