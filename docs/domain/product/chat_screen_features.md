# Функции экрана чата (Healora) 

Ниже описание основных функций экрана чата, предполагаемых взаимодействий и интеграции в текущей архитектуре проекта.

## 1. Переписка с ИИ-ассистентом
- Цель: обеспечить диалог между пользователем и ИИ-ассистентом (Healora AI).
- Архитектура взаимодействия:
  - Запрос к чат-эндпойнту: POST /api/chat
  - Тело запроса: { message: string, (опционально) profile: string }
  - Ответ: { reply: string, quiz?: { title, questions: [...] } }
- Поведение:
  - При отсутствии ключа OpenAI API возвращаются локальные/обучающие ответы (fallback).
  - Поддержка режимов: обычный чат, викторина (quiz) после ответа ассистента.
- UX/UI:
  - Беседа в чат-окне, сообщения пользователя и ИИ-болта в отдельных пузырях.
  - Возможность переключения темы и профилей через боковые панели (при наличии соответствующих UI-элементов).
- Валидация/обработка ошибок:
  - Возвращать информативные сообщения об ошибках в случае недоступности API.
- Акцептируемые данные: профили пользователя, истории чатов, журнал действий.

## 2. Натициологическое (навигационное) распознавание блюд через drag-and-drop
- Цель: предоставить метод выбора блюд из левой панели путем перетаскивания блюд в область сбора (drop zone).
- UX/UI:
  - Левая панель содержит превью блюд (мини-иконки/изображения) с подписью.
  - Пользователь может перетащить блюдо в зону сбора (drop zone). При клике на блюдо можно открыть мини-превью с подробностями.
  - В правой части может отображаться выбранное блюдо и его питательная ценность (калории, макроэлементы).
- Взаимодействие:
  - Источник блюд: массив объектов { id, name, image, calories, macros }.
  - Drag-and-Drop API: перетаскивание блюд в drop zone; drop добавляет блюдо в корзину (basket).
  - Preview: на hover или click показывать небольшое превью блюда (название, калории, макро).
- Интеграция: данные блюд могут поступать из локального набора или внешнего API.
- Акцептируемые данные: блюда, корзина блюд, превью блюд.

## 3. В правой панели настройки промптов с подсветкой персональных разделов двойника
- Цель: позволить пользователю настраивать промпты ИИ, включая персональные разделы двойника (Digital Twin).
- Функционал:
  - Блок промптов: список существующих промптов с подсветкой активных/персональных разделов.
  - Редактирование промптов: возможность редактирования текстов промптов, сохранение изменений.
  - Подсветка разделов: выделение секций, связанных с конкретными профилями/двойниками, через цветовые акценты.
- UX/UI:
  - Правая панель содержит секции "Персональные разделы двойника" с интерактивной подсветкой и контекстными подсказками.
  - Визуальные акценты: цветовая подсветка для активных разделов, соответствующая текущему профилю пользователя.
- Технические детали:
  - Промпты-хранилище: локальное состояние либо сохранение в localStorage/битовую модель на стороне клиента.
  - Возможность расширения: добавление новых секций двойника, привязка к профилям в будущем.

## Схемы взаимодействий (кратко)
- Клиент -> Бекенд: POST /api/chat (message, profile) -> ответ { reply, quiz }
- Клиент: Drag & Drop блюд из левой панели -> drop zone -> обновление basket и превью блюд
- Клиент: Правая панель — настройка промптов и подсветка персональных разделов двойника
