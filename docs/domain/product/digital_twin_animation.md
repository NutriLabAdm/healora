# Анимация цифрового двойника (Digital Twin Animation)

## Общая концепция

Анимация представляет интерактивный интерфейс для визуализации и редактирования цифрового двойника пользователя. Интерфейс разделён на три функциональные зоны:

1. **Справа** — панель входных данных (анкета, дневник, анализы, трекеры, биометрия)
2. **Сверху** — бейджи для выбора аватара (по имени/ID)
3. **Центр** — SVG-силуэт человека + таблица параметров (обновляется при выборе аватара)

---

## 1. Структура интерфейса

```
┌─────────────────────────────────────────────────────────────────┐
│  [Бейджи аватаров: ▢ Анастасия ▢ Мария ▢ Иван ▢ ... ]      │
├──────────────────────────────────┬──────────────────────────────┤
│                                  │                            │
│      SVG Силуэт человека         │    Таблица параметров      │
│      (мужской/женский)          │    (обновляется при        │
│      с индикаторами              │     выборе аватара)        │
│      (цветовая кодировка)       │                            │
│                                  │                            │
├──────────────────────────────────┴──────────────────────────────┤
│  Входные данные (справа сюда переносятся значения):           │
│  [Анкета] [Дневник] [Анализы] [Трекеры] [Биометрия]       │
└─────────────────────────────────────────────────────────────────┘
```

## 3. SVG-силуэт человека (центр)

### 3.1. Базовая структура
```
        ┌─────┐  ← Голова (импульс: ЧСС, стресс)
        │  👤  │
        └─────┘
    ╱─────│─────╲
    │      │      │  ← Руки (индикатор: активность)
    │      │      │
  ╱───────┤───────╲
  │       ТОРС      │  ← Торс (цвет: BMI, индикаторы: АД, глюкоза)
  │  (цвет: BMI)   │
  ╲───────┬───────╱
    │      │      │
    │      │      │  ← Ноги (индикатор: походка, вес)
    ╲─────┴─────╱
```

### 3.2. Цветовая кодировка силуэта (по BMI)
| BMI диапазон | Цвет | Статус |
|--------------|------|--------|
| < 18.5 | 🔵 Синий | Недостаточный вес |
| 18.5–24.9 | 🟢 Зелёный | Норма |
| 25.0–29.9 | 🟡 Оранжевый | Избыточный вес |
| ≥ 30.0 | 🔴 Красный | Ожирение |

### 3.3. Индикаторы на силуэте (точки/подсветка)
- **Голова**: пульсация по ЧСС (частота), цвет по уровню стресса
- **Грудь**: индикатор АД (сист/диаст), глюкозы
- **Живот**: индикатор окружности талии, печёночных проб (АЛТ, АСТ)
- **Руки**: уровень активности (ЧСС, тренировки)
- **Ноги**: индикатор веса, походки (шаги/день)

updateParametersTable(avatar) {

  for (const category of ['DEMO', 'BIO', 'LIFE', 'GENE', 'FAM', 'MED']) {
    const attrs = avatar.getAttributes(category);
    for (const attr of attrs) {
      const row = createTableRow(attr);
      row.status = calculateStatus(attr); // 🟢🟡🔴⚠️
      table.addRow(row);
    }
  }
}

### 5.1. Панель источников данных
Вертикальная панель с иконками-вкладками:

| Вкладка | Иконка | Содержание | Пример данных |
|---------|---------|--------------|----------------|
| **Анкета** | 📋 | Демография, анамнез, генетика, семья | Пол, возраст, курение, APOE |
| **Дневник** | 📖 | Питание, вода, сон, активность | Калории, БЖУ, шаги, сон |
| **Анализы** | 🧪 | Биохимия, гемоглобин, гормоны | Глюкоза, холестерин, ТТГ |
| **Трекеры** | ⌚ | Данные носимых устройств | ЧСС, шаги, HRV, сон |
| **Биометрия** | 📏 | Рост, вес, талия, АД | Вес 62кг, талия 72см, АД 118/76 |

### 5.2. Анимация наполнения
При выборе вкладки:
1. Справа разворачивается панель с данными
2. Данные "перетекают" (animated flow) к силуэту и таблице
3. На силуэте загораются соответствующие индикаторы
4. В таблице подсвечиваются обновлённые ячейки (жёлтая вспышка)

```javascript
// Пример анимации потока данных
animateDataFlow(sourcePanel, targetElement) {
  // 1. Создать частицы данных
  const particles = createDataParticles(sourcePanel);
  
  // 2. Анимировать движение к цели
  particles.animate({
    path: bezierCurve(sourcePanel, targetElement),
    duration: 800ms,
    easing: 'ease-out'
  });
  
  // 3. При достижении цели обновить значение
  particles.onComplete(() => {
    targetElement.updateValue();
    targetElement.flash('#FFD700', 300ms);
  });
}
```

## 6. Сценарии анимации

### Сценарий 1: Выбор аватара "Анастасия" (ID 2253)
1. Пользователь кликает на бейдж "Анастасия"
2. Бейдж подсвечивается синим
3. SVG-силуэт меняется на женский, заливается зелёным (BMI 22.8)
4. Таблица обновляется: все значения для Анастасии
5. Справа в панели "Входные данные" подсвечиваются заполненные поля
6. Индикаторы на силуэте: ЧСС 72 (спокойный пульс), АД 118/76 (норма)

### Сценарий 2: Выбор аватара "Алексей" (ID 4653, избыточный вес)
1. Пользователь кликает на бейдж "Алексей"
2. SVG-силуэт меняется на мужской, заливается оранжевым (BMI 26.8)
3. Таблица обновляется: BMI показывает 🟡 "Избыточный вес"
4. На силуэте подсвечивается живот (индикатор талии 92см > нормы)
5. В таблице БИО: глюкоза 98 (погранично), ЛПНП 135 (немного повышен)
6. Семейный анамнез: ССЗ у отца (55 лет) → индикатор риска на груди

### Сценарий 3: Обновление данных трекера (симуляция в реальном времени)
1. На вкладке "Трекеры" появляется новое значение: ЧСС = 145 (бег)
2. Частицы данных "бегут" от вкладки к голове силуэта
3. Голова силуэта начинает пульсировать красным (высокий пульс)
4. В таблице обновляется ЧСС: 72 → 145, статус меняется на 🟡
5. Через 30 минут (симуляция) ЧСС возвращается к 72, пульсация замедляется

document.querySelectorAll('.badge').forEach(badge => {
  badge.addEventListener('click', () => {
    const avatarId = badge.dataset.id;
    const avatar = avatars[avatarId];
    
    // Обновить силуэт

    // Обновить таблицу с анимацией

    // Обновить индикаторы на силуэте
    updateBodyIndicators(avatar.bio, avatar.life);

function updateSilhouette(gender, bmi) {
  const body = document.getElementById('body');
  const silhouetteFile = gender === 'F' ? 'silhouette-female.svg' : 'silhouette-male.svg';
  
  // Цвет по BMI
  let color;
  if (bmi < 18.5) color = '#2196F3';      // синий
  else if (bmi < 25) color = '#4CAF50';     // зелёный
  else if (bmi < 30) color = '#FF9800';     // оранжевый
  else color = '#F44336';                   // красный
  
  body.setAttribute('fill', color);
}

function updateTableWithAnimation(avatar) {
  const tbody = document.getElementById('twin-table-body');
  tbody.innerHTML = '';
  
  // Заполнить таблицу с задержкой для эффекта "печатания"
  const allRows = prepareRows(avatar);
  allRows.forEach((row, index) => {
    setTimeout(() => {
      const tr = document.createElement('tr');
      tr.innerHTML = `<td>${row.category}</td><td>${row.attr}</td><td>${row.value}</td><td>${row.status}</td>`;
      tr.style.opacity = '0';
      tr.style.transform = 'translateX(-20px)';
      tbody.appendChild(tr);
      
      // Анимация появления
      requestAnimationFrame(() => {
        tr.style.transition = 'all 0.3s ease';
        tr.style.opacity = '1';
        tr.style.transform = 'translateX(0)';
      });
    }, index * 50); // по 50мс на строку
  });
}

## 8. Визуальные эффекты

| Эффект | Описание | Применение |
|--------|----------|------------|
| **Flow particles** | Частицы данных "перетекают" от источника к силуэту | Обновление данных трекеров |
| **Pulse** | Пульсация индикатора (голова = ЧСС, талия = риск) | Отображение витальных функций |
| **Color sweep** | Плавное изменение цвета силуэта (BMI) | Смена аватара |
| **Flash** | Жёлтая вспышка ячейки таблицы | Обновление значения |
| **Progress bar** | Заполнение шкалы (вода, шаги, сон) | Дневник и трекеры |
| **Alert shake** | Тряска индикатора при критическом значении | АД > 140/90, глюкоза > 126 |
